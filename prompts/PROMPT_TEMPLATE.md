# 🤖 PROMPT_TEMPLATE.md

## AI工程调用模板库

> 用于规范 AI 使用方式，避免失控

---

# 🧠 使用原则

* 不写“聊天式prompt”
* 只用“工程化prompt”
* 每次调用都有结构

---

# 🧩 1. 标准开发 Prompt（最常用）

```text
你是一个资深后端工程师（Python + FastAPI）

【任务】
实现一个模块：<模块名称>

【上下文】
<相关代码 / 文档摘要>

【契约】
<API契约 + DB结构>

【技术要求】
- 使用FastAPI
- 使用SQLAlchemy
- 遵循现有项目结构

【约束】
1. 只实现当前模块
2. 不修改其他文件
3. 必须返回完整代码
4. 不写伪代码
5. 必须可运行

【输出格式】
1. 文件路径
2. 完整代码
```

---

# 🧩 2. Debug Prompt

```text
你是资深工程师

【问题】
<报错信息>

【代码】
<相关代码>

【要求】
1. 找出问题根因
2. 给出修改方案
3. 提供修复后的完整代码

不要泛泛解释
```

---

# 🧩 3. Refactor Prompt

```text
请对以下代码进行重构：

目标：
1. 提升可读性
2. 减少重复逻辑
3. 保持功能不变

输出：
- 重构后代码
- 修改说明
```

---

# 🧩 4. Review Prompt（重要）

```text
作为架构师 review 以下代码：

重点：
1. 模块职责
2. 代码重复
3. 潜在bug
4. 扩展性问题

输出：
- 问题列表
- 优化建议
```

---

# 🧩 5. API设计 Prompt

```text
设计一个API：

需求：
<业务描述>

输出：
1. 路径
2. 方法
3. 请求参数
4. 返回结构
```

---

# 🔥 最重要原则

## ❗ 永远限制AI

* 限制范围
* 限制输出
* 限制行为

---

## ❗ Prompt必须包含

* 角色
* 任务
* 上下文
* 契约
* 约束
* 输出格式

# 🧠 一句话总结
