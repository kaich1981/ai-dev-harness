# 🧠 AI_DEV_SOP.md

## AI 辅助个人开发标准流程（Harness Engineering 版）

> Build AI products with control, not chaos.

---

# 🚀 0. 核心理念（必须先理解）

AI不是开发者，你才是。

> ❌ Prompt → Code → Chaos
> ✅ System → Contract → Task → Controlled AI Execution

---

## 🎯 你在项目中的角色

| 角色   | 你  | AI |
| ---- | -- | -- |
| 架构设计 | ✅  | ❌  |
| 任务拆分 | ✅  | ❌  |
| 代码实现 | ⚠️ | ✅  |
| 验收控制 | ✅  | ❌  |

---

# 🧭 1. 全流程总览

```
Idea
 ↓
Requirement (需求)
 ↓
Architecture (技术方案)
 ↓
Contracts (契约) ⭐
 ↓
Task Breakdown (任务拆分)
 ↓
AI Execution (逐任务开发)
 ↓
Validation (验收)
 ↓
Iteration (迭代)
```

```markdown
-
```

# 🔥 4. 阶段 3：契约定义（核心控制层）

> 没有契约，不允许写代码

---

## 4.1 数据库契约

```sql
Table: xxx

id INT PK
field_name TYPE COMMENT
```

---

## 4.2 API契约

```json
POST /api/xxx

Request:
{}

Response:
{}
```

---

## 4.3 AI输出契约

```json
{
  "field": "string",
  "confidence": "0-1"
}
```

---

## 4.4 项目结构契约

```
app/
 ├── api/
 ├── service/
 ├── models/
 ├── schemas/
 ├── utils/
```

---

# ⚙️ 5. 阶段 4：任务拆分（决定成败）

## 🎯 原则

* 每个任务 ≤ 1小时
* 必须可验证
* 不跨模块

---

## 📄 标准任务模板

```markdown
# 任务名称
-

# 输入
-

# 输出
-

# 约束
-

# 验收标准
-
```

---

# 🤖 6. 阶段 5：AI执行规范

## ❌ 禁止

* 一次生成整个项目
* 修改未授权模块

---

## ✅ AI执行模板（核心）

```text
你是后端工程师

【任务】
实现XXX模块

【上下文】
（相关文档）

【契约】
（API + DB结构）

【要求】
1. 只实现当前任务
2. 不修改其他模块
3. 返回完整可运行代码
4. 不写伪代码

【输出】
文件路径 + 代码
```

---

# ✅ 7. 阶段 6：验收机制

## 必须全部通过

---

### 7.1 运行验证

* 服务能启动
* 接口可调用

---

### 7.2 契约验证

* JSON结构正确
* 字段一致

---

### 7.3 真实实现

* 无 mock
* 无 TODO

---

### 7.4 副作用检查

* 未修改无关模块

---

# 🔁 8. 阶段 7：回归机制

---


---

## 必须保留测试用例

# 🧠 14. Harness Engineering 精髓

---

> AI的价值不在“写代码”
> 而在“在你的系统中稳定工作”

---

## 你真正构建的是：

* Prompt系统
* 任务系统
* 验收系统
* 控制系统

# 🏁 最终结论

> 不要让 AI 写项目
> 要让 AI 在你的系统中完成任务
