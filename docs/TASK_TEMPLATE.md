# 🧩 TASK_TEMPLATE.md

## AI任务驱动开发模板

> 用于拆分任务并驱动 AI 精确执行

---

# 🎯 使用方式

每次开发一个功能，必须先创建一个任务，然后再调用 AI。

---

# 📄 标准任务模板

```markdown
# 🧩 Task: <任务名称>

## 1. 背景
说明这个任务解决什么问题（1-2句话）

---

## 2. 输入（必须明确）
- 需求文档：
- API契约：
- 数据库结构：
- 相关模块：

---

## 3. 输出（必须明确）
本任务必须产出：

- 文件路径：
  - app/api/xxx.py
  - app/service/xxx_service.py

- 功能：
  - 实现xxx接口
  - 完成xxx逻辑

---

## 4. 约束（非常重要）
- 不允许修改其他模块
- 必须符合API契约
- 必须使用已有数据库连接
- 返回结构必须一致

---

## 5. 验收标准（必须可验证）

### 功能
- [ ] 接口可调用
- [ ] 返回正确JSON结构

### 技术
- [ ] 无报错
- [ ] 无TODO / mock

### 逻辑
- [ ] 满足业务需求

---

## 6. 测试用例

输入：
```

{
"xxx": ""
}

```

预期输出：
```

{
"xxx": ""
}

````

---

## 7. AI执行指令（直接复制用）

```text
你是后端工程师

【任务】
<任务名称>

【上下文】
<粘贴必要文档>

【契约】
<API + DB结构>

【要求】
1. 只实现当前任务
2. 不修改其他模块
3. 输出完整代码
4. 不写伪代码

【输出格式】
- 文件路径
- 代码
````

```

---

# 🔥 核心原则

- 一个任务只做一件事
- 每个任务必须可验证
- 每个任务必须能独立运行
- 不允许“顺便改别的”

---

# ❗ 常见错误

- 任务过大（>1小时）
- 输出不明确
- 没有验收标准
- AI随意扩展
```
